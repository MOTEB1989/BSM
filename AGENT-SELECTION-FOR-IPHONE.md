# دليل اختيار الوكلاء من iPhone (BSM)

هذا الدليل يوضح **لماذا** تختار وكيلًا معينًا حسب المهمة، خاصة عند استخدام الهاتف (iPhone) بدل الكمبيوتر.

## 1) الوكلاء المناسبة للاستخدام المباشر (Selectable)

| Agent ID | Name | الأفضل له | لماذا هذا الوكيل؟ |
|---|---|---|---|
| `ios-chat-integration-agent` | iPhone Chat Integration Agent | دعم تكامل المحادثة على الهاتف، فحص تجربة الاستخدام عبر mobile/web chat | لأنه مخصص لتكامل iPhone ومتاح في `chat/api/mobile` ومخاطره منخفضة. |
| `governance-agent` | Governance Agent | الاستشارات العامة للحوكمة والامتثال | مناسب للحوارات الآمنة بدون تعديلات نظامية مباشرة. |
| `legal-agent` | Legal Analysis Agent | الأسئلة القانونية/السياسات بصيغة استشارية | Agent استشاري منخفض المخاطر ومتاح للمحادثة. |

## 2) وكلاء داخلية (Internal) — غالبًا CI/GitHub

> هذه الوكلاء قوية، لكنها عادة ليست مخصصة للاختيار المباشر من واجهة المستخدم.

| Agent ID | Name | نوع الاستخدام | ملاحظة |
|---|---|---|---|
| `governance-review-agent` | Governance Compliance Officer | مراجعة التوافق داخل CI/GitHub | `internal_only: true` و `selectable: false`. |
| `code-review-agent` | Code Review Agent | مراجعات كود آلية في CI | يعمل كمدقق وليس وكيل محادثة مباشر للمستخدم النهائي. |
| `security-agent` | Security Agent | تدقيق أمني وتقارير ثغرات | Agent تدقيقي آمن، موجه لخطوط CI/GitHub. |
| `pr-merge-agent` | PR Merge Agent | الدمج الآلي للـ PR بعد الحوكمة | يتطلب موافقات آلية ووضع تشغيل مقيّد. |

## 3) قرار سريع (Decision Tree)

- إذا كنت تريد **محادثة من الهاتف** عن تجربة المستخدم/التكامل: استخدم `ios-chat-integration-agent`.
- إذا تريد **سياسات/حوكمة**: استخدم `governance-agent`.
- إذا تريد **تحليلًا قانونيًا/امتثال نصي**: استخدم `legal-agent`.
- إذا تريد **مراجعة أمنية/كود/دمج PR**: فعّل بايبلاين CI ودع الوكلاء الداخلية تعمل تلقائيًا.

## 4) أفضل ممارسة تشغيل من iPhone

1. افتح GitHub Codespaces من Safari.
2. شغّل المهام التطويرية داخل Codespace.
3. استخدم الوكلاء `selectable` للأسئلة المباشرة.
4. اترك مراجعات الأمن/الكود/الدمج للوكلاء الداخلية عبر CI.

## 5) Why (لماذا هذا التقسيم مهم؟)

- يقلل التعقيد على الهاتف عبر فصل: **محادثة مباشرة** vs **أتمتة داخلية**.
- يمنع استخدام وكيل غير مناسب لسياق المستخدم النهائي.
- يرفع الجودة: الوكلاء الداخلية تعمل داخل CI مع تتبع ومراجعة أفضل.
